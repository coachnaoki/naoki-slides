# 記述ルール（全テンプレート共通）

`templates/slides.html` の冒頭コメントに定義されたルールを集約。JSON を組み立てる時は必ずこれに従う。

## 1. テキスト記述ルール

1. **1項目につき最大2行まで**
2. **改行は文節の境界で行う**（助詞・て形・文末の直後など）
3. **改行直前に「、」「。」を付けない**（削除する）
4. **単語の途中で改行しない**
5. **1行に収まる文には改行を入れない**

### 🚫 絶対NG: 長い文の扱い

スライドのタイトル・キャッチフレーズ等で**文が長くてブロックに被る／崩れる**場合：

- ❌ **改行を入れて2行にする** — NG
- ❌ **font-size を小さくして縮める** — NG（文字は絶対に小さくしない）
- ✅ **文を短く書き換える** — 唯一の正解

例：
- `前衛が覚えるべき3つの鉄則` → `前衛の3つの鉄則`
- `覚えるだけで勝率が変わる` → `勝率が跳ねる`

台本からキーワード・数字を残しつつ、修飾語を削って短文化する。
短縮案は**必ずユーザーに確認してから**適用する（勝手な言い換え禁止）。

## 2. 改行タグの使い分け

テンプレートごとに改行記号が違う。**混同すると表示が崩れる**ので注意。

| 書き方 | 使うテンプレート・フィールド |
|---|---|
| `<br>` | `steps` の `content` / `label`、`closing` の `desc`、`big-message` の panel の `label` |
| `\n` | `big-message` の `message`、`title` の `title`（JSで `<br>` に自動変換） |

### 例

```json
// steps（<br>を使う）
{ "type": "steps", "steps": [
  { "num": 1, "label": "初動", "content": "頭の中で描いたことを「2回目」に<br>実行する場所にすぎない" }
]}

// big-message（\nを使う）
{ "type": "big-message", "message": "勝敗を分けるのは\nどちらの試合を制するかだ" }
```

## 3. 黄色マーカーの書き方

```html
<span style='background:#CCFF00'>強調したいテキスト</span>
```

- **`px-2` などのパディングは付けない**（隣の文字に被るため）
- タイトル内の特定語を強調する時に使う（例: "前衛の`<span>`教科書`</span>`"）

## 4. title テンプレートの注意点

- `title`（1行目）と `titleHighlight`（2行目の黄色ハイライト部分）に**同じ文章を入れない** → 重複表示
- `title` の末尾に「、」を付けない（次行との間の改行直前になるため）
- `titleSuffix` = 2行目のハイライト後に続く非ハイライト文字列（任意）

### 例

```json
{
  "type": "title",
  "icon": "fa-trophy",
  "title": "ダブルスで勝つための",
  "titleHighlight": "前衛の教科書",
  "titleSuffix": "",
  "subtitle": "知識と戦術で相手を圧倒する方法"
}
```

## 5. steps テンプレートのスタイル

各ステップに `style` プロパティを付けて色分けする。

| style値 | 見た目 | 用途 |
|---|---|---|
| `"green"` | 緑ボーダー | 通常の重要項目 |
| `"gray"` | グレー | 補足・やや重要度が低い項目 |
| `"highlight"` | 黄緑背景 | 最重要・まとめ |

## 6. 全体のカラールール

- メインカラー（黄緑）: `#CCFF00`
- ダーク: `#121212`
- フォント: `Zen Kaku Gothic New`（weight 700/900）

これらの値は既に `templates/slides.html` に埋め込まれているため、JSON 側で色を指定する必要はない。
