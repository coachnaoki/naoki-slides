Naoki式 AI スライド自動生成テンプレート

台本テキストを渡すだけで Google Slides が完成する naoki-slides のセットアップガイド

⚠ 前提：naoki-blueprint 生徒限定

本テンプレは naoki-blueprint 生徒限定です。ライセンスIDは naoki-blueprint で発行されたものをそのまま使います（新規に ID は発行しません）。

naoki-blueprint を未導入の方は、まず naoki-blueprint のセットアップ を先にお願いします。

1 リポジトリをダウンロード

⚠ 事前準備: すでに Claude Code を開いている方は、ターミナル上で Ctrl+C（Mac は Control+C）を 2回押して終了してから進めてください。

Cursor のターミナルで、naoki-slides を置きたいフォルダに移動します（naoki-blueprint と同じ階層に置くのが推奨・自動でライセンスを引き継ぎできます）。

Mac の場合

cd ~/Desktop/Cursor

Windows (PowerShell) の場合

cd ~\Desktop\Cursor

そのフォルダでリポジトリをダウンロード（Mac / Windows 共通）:

git clone https://github.com/coachnaoki/naoki-slides.git
cd naoki-slides
npm install

2 ライセンス認証

naoki-blueprint で発行されたライセンスID をそのまま使います（新規 ID は発行されません）。

🚀 おすすめ: naoki-blueprint から自動コピー

naoki-blueprint で既に認証済みなら、ID を打ち直さずに自動コピーできます：

node tools/find-license.mjs

同じ PC で naoki-blueprint/.license が見つかれば自動コピー。ID は打たなくて OK。

手動で認証する場合

node tools/validateLicense.mjs NK-XXXX-XXXX-XXXX

認証に成功すると ✓ {あなたの名前} さん、認証済みです と表示されます。

PC を変更する場合は Chatwork で Naoki に個別メッセージしてください。

3 セットアップスクリプトを実行

Google Slide テンプレと GAS（書き込み用スクリプト）を自分の Drive にセットアップします。

Mac の場合

./セットアップ.sh

Windows (PowerShell) の場合

bash セットアップ.sh

対話形式で以下を順番に案内されます：

Step 3-1. Slide テンプレを自分の Drive にコピー

  1. 画面に表示されたリンクを開く（「コピーを作成」ボタンが自動で表示）
  2. 「コピーを作成」 を押す → 自分の Drive に Slide + GAS スクリプトが一緒にコピーされる
  3. コピー後に開いた Slide の URL をそのままコピー
  4. ターミナルに URL を貼り付けて Enter

Step 3-2. GAS をデプロイして URL を取得

  1. Step 3-1 でコピーした Slide を開いたまま、上部メニュー 「拡張機能」→「Apps Script」 をクリック → 新しいタブでスクリプトエディタが開く
  2. 画面右上の 「デプロイ」→「新しいデプロイ」
  3. 歯車アイコン → 「ウェブアプリ」 を選択
  4. 次のユーザーとして実行: 「自分」
  5. アクセスできるユーザー: 「全員」（「自分のみ」だと CLI から叩けません）
  6. 「デプロイ」 を押す
  7. 初回は承認ダイアログ：「詳細」→「{プロジェクト名}（安全ではないページ）に移動」→ 許可
  8. 表示された 「ウェブアプリ URL」 をコピー
  9. ターミナルに URL を貼り付けて Enter
なぜ「全員」？ CLI からリクエストを送るため。URL は推測困難なので、セキュリティ上は問題ありません。
完了すると .env が自動生成されます。以降は触らなくて OK。

4 新規プレゼンを作成

ここから毎回の作業。プレゼン1本ごとに専用フォルダを作ります。

Mac の場合

./新規スライド.sh

Windows (PowerShell) の場合

bash 新規スライド.sh

プレゼン名を聞かれるので入力（例: tennis-camp-2026）。
→ projects/tennis-camp-2026/script/台本.md が作られ、自動でエディタで開きます。

5 台本を書く

開かれた script/台本.md に自由に書きます。話し言葉・箇条書き・見出し混在でOK。

ファイル形式は自由：.md じゃなくても .txt や他の拡張子でも動きます。script/ フォルダに台本ファイルが1つ入っていればOK。

Claudeが台本を解析 → 17種類のテンプレート（タイトル／3カード／ステップ／対比／統計 等）から最適なものを自動選択してスライド化します。

6 Claude Code でスキル起動

プレゼンフォルダに移動して Claude Code を起動：

cd projects/tennis-camp-2026
claude --dangerously-skip-permissions

Claude Code が立ち上がったら、以下を入力：

/slides-create

自動で以下の流れが走ります：

  1. 台本を解析してスライド構成を設計
  2. 構成案を提示 → あなたが OK と言うまで JSON 確定しない
  3. Puppeteer で画像化（1280×720 PNG × N枚）
  4. プレビューをブラウザで自動表示 → 目視確認
  5. OK なら GAS 経由で Google Slide に反映
  6. 完成 URL が返ってくる → ブラウザで開いて確認
約1分で完成。編集したい場合は script/ 内の台本を修正して再実行してください。

アップデート（バージョンアップ）

テンプレートが更新されたら、以下のコマンドで最新版に置き換えできます。
あなたが作った projects/ と .license / .env は上書きされず安全です。

Mac の場合

cd ~/Desktop/Cursor/naoki-slides
./アップデート.sh

Windows (PowerShell) の場合

cd ~\Desktop\Cursor\naoki-slides
bash アップデート.sh

裏側で走っている処理（参考）

フェーズ1: 設計
台本解析Claude が章立て・数字・キーワードを抽出
テンプレート選択17種から最適なテンプレを自動選択
構成案提示あなたに OK を貰う（必須）
↓
フェーズ2: 生成
JSON 生成slide-data.json を書き出し
Puppeteer 画像化HTMLテンプレを 1280×720 PNG にキャプチャ
プレビュー表示ブラウザで全枚数を縦並びに表示
↓
フェーズ3: 反映
GAS へ POSTBase64化した画像を Webhook で送信
Slide 書き換えGAS が既存スライドを全削除 → 画像を貼り付け
完成Google Slide の URL が返る

17種類のテンプレート（自動選択）

Claude が台本の内容から適切なテンプレートを自動選択します。

テンプレート用途
title表紙・タイトルページ
agendaアジェンダ・目次
sectionセクション区切り
three-cards3つのポイント・特徴
three-tactics3つの戦術・役割
two-columns2つの対比・比較
steps手順・ステップ
big-message重要メッセージ・結論
before-afterBefore/After
stats統計・数字
checklistチェックリスト
timeline時系列・流れ
rankingランキング
versus対立・比較
highlight-box強調ボックス
quote引用・名言
closing締め・CTA

フォルダ構成

naoki-slides/                     ← ダウンロードしたフォルダ
├── README.md                     ← ガイド
├── セットアップ.sh                 ← 初回セットアップ（.env 自動生成）
├── 新規スライド.sh                 ← 新規プレゼン作成
├── アップデート.sh                 ← 最新版に更新
├── .claude/skills/slides-create/  ← メインスキル
├── .template-project/             ← 新規プレゼンの雛形
│   └── script/台本.md
├── templates/slides.html          ← 17種テンプレ
├── tools/                         ← Node/JSユーティリティ
│   ├── validateLicense.mjs
│   ├── find-license.mjs
│   ├── screenshot-slides.mjs
│   ├── generate-preview.mjs
│   └── post-to-gas.mjs
├── gas/                           ← GAS本体（参考）
│
└── projects/                      ← あなたのプレゼン（gitignore）
    └── {プレゼン名}/
        ├── script/台本.md          ← 台本（形式自由）
        ├── slide-data.json         ← Claudeが生成
        └── output/                 ← スライド画像
